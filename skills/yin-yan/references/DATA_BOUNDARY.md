# 数据边界

## 默认原则

完整账本保存在用户本地 `ledger.json`。测试阶段不回收完整账本。

skill 可以读取和写入本地账本文件，但不能自动读取微信、支付宝、银行流水、系统通知、支付凭证截图或全量聊天记录。

## 本地防丢账

- 每次写入 `ledger.json` 前，先在同目录 `backups/` 下保存一份最近账本备份。
- 每条自然语言输入会追加到 `raw-events.ndjson`，用于账本异常后的人工重建。
- 写账本时使用临时文件校验后替换，避免中途崩溃留下半截 JSON。
- 如果记录数会从非 0 变成 0，或普通写入导致记录数减少，脚本会拦截并提示先查看备份。
- 查看备份：`查看备份` 或 `/查看备份`。
- 恢复最近备份：`恢复最近备份` 或 `/恢复备份`。
- 清空账本必须二次确认：先发送 `清空账本` 查看风险提示，再发送 `确认清空账本` 才会清空。清空前仍会自动备份。
- 本地最多保留最近 50 份账本备份。

## 不收集

- 完整 `ledger.json`
- 微信、支付宝、银行原始账单
- 银行卡号、身份证号、手机号
- 支付凭证截图
- IM 全量聊天记录
- 真实客户、项目或家庭隐私

## 可回收

- `/测试报告` 生成的匿名摘要
- 用户手动脱敏后的 `/分享错误` 样例
- 主观体验反馈

## 解释口径

所有汇总、预算进度、提醒都必须表达为“仅基于已记录数据”，不能暗示已经覆盖用户全部真实财务。

现金流心跳必须进一步说明“仅基于已记录数据和你手动提供的余额估算”。
