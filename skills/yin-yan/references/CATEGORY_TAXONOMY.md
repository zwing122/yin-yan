# 类目体系

## 目标

银砚使用两级类目，避免把不同粒度的概念混在一起。一级类目用于总览和预算大盘，二级类目用于实际记账、提醒和复盘。

保留旧字段 `category` 作为兼容字段，默认等于 `category_l2`。新逻辑应优先读取 `category_l1` 和 `category_l2`。

## 基础结构

| 一级类目 | 二级类目示例 | 说明 |
| --- | --- | --- |
| 餐饮 | 正餐、外卖、快餐、小吃、咖啡、茶饮、饮料、零食饮料 | 日常吃喝，二级类目必须尽量细 |
| 交通 | 打车、公共交通、加油、停车、火车机票 | 出行相关 |
| 居住 | 房租、物业、水电燃气、维修、家政 | 居住固定或维护支出 |
| 购物 | 日用品、服饰、美妆、数码、家居 | 实物消费 |
| 医疗 | 药品、门诊、体检、保险 | 健康相关 |
| 娱乐 | 影音会员、游戏、演出、旅行玩乐 | 非刚需体验消费 |
| 人情 | 红包、礼物、请客、份子钱 | 社交关系支出 |
| 学习 | 课程、书籍、工具、考试 | 学习成长 |
| 收入 | 工资、奖金、兼职、报销、退款 | 入账类 |
| 转账 | 账户转移、还款、借出、借入 | 资金移动，不等同消费 |

## 餐饮细分

- `餐饮 / 正餐`：午饭、晚饭、吃饭、餐厅、食堂、聚餐。
- `餐饮 / 外卖`：美团外卖、饿了么、点外卖、外卖订单。
- `餐饮 / 快餐`：麦当劳、肯德基、汉堡、炸鸡等快餐语义。
- `餐饮 / 小吃`：夜宵、炸串、煎饼、烧烤、路边摊等。
- `餐饮 / 咖啡`：咖啡、瑞幸、库迪、星巴克、Manner、Tims 等。
- `餐饮 / 茶饮`：奶茶、喜茶、奈雪、霸王茶姬、茶百道、蜜雪冰城等。
- `餐饮 / 饮料`：水、可乐、饮料、果汁等。
- `餐饮 / 零食饮料`：零食、薯片、糖果、便利店零食等。

## 商户词线索

商户词、店名、商品名只用于帮助归类，不能成为独立维度。

允许：

```text
瑞幸18 -> 餐饮 / 咖啡
库迪咖啡 12 -> 餐饮 / 咖啡
喜茶 25 -> 餐饮 / 茶饮
美团外卖 38 -> 餐饮 / 外卖
```

禁止：

```text
brand: 瑞幸
merchant: 瑞幸
预算 scope: brand
输出“品牌：瑞幸”
```

## 不确定分类

当金额明确但分类不确定时，先记录事实并标记审查，不要追问过多。

示例：

```text
买东西50 -> 记录金额，分类可进入待审查
在小王那花了80 -> 不确定是餐饮、购物还是人情，进入待审查
```

低打扰追问只问最小必要问题，例如：“这笔更像餐饮还是购物？”
