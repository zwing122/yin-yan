# 预算规则

## 定位

第一版只做轻量预算提醒，不做完整预算系统。

## 自然语言入口

```text
这个月外卖别超过 800
这个月咖啡预算 300
咖啡每天最多 2 笔
外卖一天最多 1 单
交通这个月控制在 500
娱乐预算 500
```

这些输入记录为 `goal` 事件，并归到对应预算范围。预算范围只支持一级类目和二级类目；第一版默认优先落到二级类目，例如“咖啡”不是粗略归到“餐饮”，而是 `餐饮 / 咖啡`。商户词只作为分类线索，不作为预算维度。

预算限制类型：

- 金额月预算：例如“咖啡这个月别超过 300”。
- 每日笔数预算：例如“咖啡每天最多 2 笔”“外卖一天最多 1 单”。

## 预算进度条

今日、本周、本月汇总和现金流心跳都必须可以输出预算进度：

```text
🍱 外卖预算：¥620 / ¥800 ████████░░ 78%（剩余 ¥180）
🍱 咖啡每日笔数：3 / 2 ██████████ 150%（超出 1 笔）
```

必须说明预算进度仅基于已记录数据，不代表完整真实消费。

## 预算提醒

heartbeat 中输出：

```text
预算提醒：
- 外卖预算接近上限：720/800
- 咖啡今日笔数已超：3/2
- 娱乐预算已超：560/500
```

默认阈值：

- 已用 80% 以上：接近上限
- 已用 100% 以上：已超预算
- 每日笔数达到上限：提示已到上限
- 每日笔数超过上限：提示已超，并说明超出几笔

## 不做

- 年度预算
- 预算滚存
- 家庭多人预算
- 多账户预算体系
- 自动分摊
- 会计级预算报表
