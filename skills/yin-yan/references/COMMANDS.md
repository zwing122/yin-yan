# 命令和自然语言入口

第一版不要求用户记忆命令。`/命令` 只作为兼容和调试入口。

## 回复约束

只发送最终结果或最小确认问题。不要在 IM 里输出以下内容：

```text
任务准备中，正在为您智能调度算力...
调度完成，正在生成回复...
我来直接修复这 4 条事项
现在记三条固定支出
```

正确风格：

```text
需要确认：物业费是哪一天到期？
已记录 3 条固定支出。
本月复盘（仅基于已记录数据）...
```

## 画像输入

这些输入不要求初始化配置，会在自然记录中逐步沉淀现金流画像：

```text
每月 10 号发工资 12000
微信余额 200
银行卡余额 3200
每月 5 号房租 2500
花呗 20 号还，通常 800 左右
这个月外卖别超过 800
```

## 批量输入

每行一条：

```text
今天早餐 12，微信付的
今天午饭 38，支付宝付的
每月 15 号手机套餐 129
```

分号分隔：

```text
今天早餐 12，微信付的；今天午饭 38，支付宝付的；每月 15 号手机套餐 129
```

## 新手菜单

用户发送以下内容时，只返回帮助菜单，不写入账本：

```text
帮助
菜单
功能菜单
新手引导
使用说明
怎么用
如何使用
```

菜单应包含自然语言示例、复盘入口、待复核/备份/测试报告入口，以及定时提醒由宿主或本机计划任务按用户设置触发的说明。

如果 `ledger.json` 不存在、为空文件，或 `events` 为空，帮助菜单后必须追加一段“第一次使用建议：先设自动提醒和汇总”，包含可复制给 OpenClaw/Hermes/QClaw 的自动提醒提示词。

## 高歧义样例

详细复核规则见 `RECORDING_RULES.md`。这些应进入待复核：

```text
招商卡还信用卡 3000，这不是消费
微信提现 1000 到银行卡
客户退了 500
我先垫付 600，后面报销
我午餐12块钱
明天需要付电费50
```

期望追问：

```text
我午餐12块钱 -> 需要确认付款方式
明天需要付电费50 -> 需要确认一次性还是每月/固定周期
```

## 补录和余额

这些不要求导入账单，都是用户手动补一句：

```text
昨天电费50，微信付的
前天停车费20，支付宝付的
微信余额 200
招商卡余额 3500
```

heartbeat 可以基于已记录数据提示：

```text
补录建议：
- 历史项目：之前记录过房租¥2500，本期如已发生可补一句“本月房租2500已付”。
- 过去账单：如果昨天或前天有漏记支出/缴费，可以直接补一句“昨天电费50，微信付的”。
- 账户余额：建议每周补一次主要账户余额，例如“微信余额 200”。
```
