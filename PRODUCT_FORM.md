# 产品形态检查

## 第一技术存在形式

第一版是 OpenClaw/Hermes 中的个人财务助理 skill pack。

它不是独立 App，不是完整 SaaS，也不是自动记账工具。

## 长期产品形态

长期可以演化为混合形态：

- OpenClaw/Hermes skill pack
- 本地账本内核
- 本地网页复盘器
- 可选托管解析接口
- 后续创业者版 API/SDK 或私有化模块

## 宿主平台依赖

第一版依赖 OpenClaw/Hermes 提供：

- 用户已有聊天/IM 入口
- agent 运行时
- skill 加载机制
- cron/heartbeat 或定时任务能力
- 本地文件读写环境
- 用户自己的模型配置或 API key

## 产品自己承担

银砚 skill pack 自己承担：

- `SKILL.md` 行为说明
- 财务事件 schema
- 本地 `ledger.json` 读写契约
- 自然语言解析输出契约
- 高歧义事件复核规则
- 心跳摘要模板
- 匿名 `test_report` 生成契约
- QQ 群测试材料
- 本地网页导入和图表复盘契约

## 不承担

第一版不承担：

- 云托管 OpenClaw/Hermes
- 托管用户 IM 凭证
- 自动读取微信、支付宝或银行流水
- 自动支付、转账、催收或投资执行
- 完整账号体系和订阅计费系统
- 多平台大规模分发
- 服务端保存完整账本

## 对数据的影响

完整账本默认留在用户本地。

测试阶段只回收：

- 用户复制的匿名 `/测试报告`
- 可选 `test_report.json`
- 用户自愿分享的脱敏解析错误样例

不回收：

- 完整 `ledger.json`
- 原始支付流水
- 银行卡、身份证或真实支付凭证
- 全量聊天记录

## 对收费的影响

第一版不依赖 OpenClaw/Hermes 直接收费。

可选商业化只围绕：

- 个人端低价 token 或托管解析额度
- BYOK / 本地优先模式
- 后续创业者版 API、SDK、规则包、私有化部署

## 对验证的影响

验证不应先做网站或 SaaS。

验证顺序：

1. 固定本地账本和匿名报告 schema。
2. 实现最小 skill 行为。
3. 用 QQ 群做 7 天测试。
4. 收匿名报告和脱敏错误样例。
5. 再决定是否做本地网页复盘器或托管解析接口。
